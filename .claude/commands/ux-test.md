Test the UX of the implemented feature: $ARGUMENTS

Imagine you are a sales rep using this feature.

## Test Scenarios
Generate and mentally walk through:
1. Happy path - normal usage
2. Error path - what happens when things fail
3. Edge cases - empty states, large data, slow network
4. Discovery - how do users find this feature
5. Recovery - how do users fix mistakes

## For Each Scenario
- **User Action**: What they do
- **Expected Result**: What should happen
- **Feedback**: What tells them it worked
- **Failure Mode**: What could go wrong

## Accessibility Check
- Keyboard navigation works
- Screen reader friendly
- Color contrast sufficient
- Loading states present

## Output
Prioritized list of UX issues with specific fixes.
